### はじめに

Web サービスは、1 人の登場人物で動いているわけではありません。  
いくつかの役割を持った技術要素（しくみ）が、連携しながら動いています。

このセクションでは、Web サービスの「裏側でどんなやり取りが行われているのか」を  
わかりやすく、順を追って見ていきます。

---

### 例：ネットショッピングで「イヤホン」を検索したとき

あなたが通販サイトで「イヤホン」と入力して検索ボタンを押したとします。  
すると、Web の中ではこんな流れで処理が進みます。

---

### やり取りの流れ（シンプル版）

1. あなた（ユーザー）が「イヤホン」と検索
2. ブラウザ（スマホや PC の画面）が、検索のリクエストを送信
3. Web サーバーが、そのリクエストを受け取る
4. アプリケーションサーバーが、検索の処理を行う（どんなイヤホンがあるか探す）
5. データベース（DB）が、イヤホンに関するデータを返す
6. アプリケーションサーバーが、そのデータを元に検索結果の画面を作る
7. Web サーバー → ブラウザへ画面を返す
8. ブラウザが画面を表示し、あなたは検索結果を見る

---

### 実際にはこんなやり取りをしている

| 登場人物                 | セリフのように言うと？                                   |
| ------------------------ | -------------------------------------------------------- |
| ブラウザ                 | 「検索したいみたいなので、Web サーバーさんお願いします」 |
| Web サーバー             | 「了解。アプリ側に処理を依頼します」                     |
| アプリケーションサーバー | 「検索ワードもらったので、DB にデータを聞いてみよう」    |
| データベース（DB）       | 「はい、イヤホンのデータはこちらです」                   |
| アプリケーションサーバー | 「じゃあ、見やすい形に整えて返しますね」                 |
| Web サーバー             | 「ブラウザさん、これをユーザーに表示してください」       |

---

### 技術用語を少しだけ

Web のやり取りは、「リクエスト」と「レスポンス」という言葉で説明されます。

- リクエスト：ブラウザから「これが見たい」「これを送る」と要求すること
- レスポンス：それに対して「これが結果です」と返すこと
